The Perl Audio Converter 🇬🇧 is a powerful tool for converting audio files and can also be used to extract audio data from video file formats. A large number of also quite "exotic" formats are supported. The operation takes place via the command line, under KDE the Perl Audio Converter can be integrated in Konqueror and Dolphin . There are also connections for the file managers Nautilus , Nemo and Thunar .

For some formats, an existing tag can be read out and created for the new format. The program can also be used to rip CDs and save them in the supported formats. The article Converting Audio Files describes a variety of other conversion options. However, the Perl Audio Converter achieves top scores in terms of the range of supported formats.

Installation¶

The following packages must be installed [1] :

  • pacpl (universe)

Package list to copy: 

sudo apt-get install pacpl

Or install with apturl , Link: apt: // pacpl

Manual Installation

In order to create the program in the current version 6.1.0 (as of April 2019) from the source code, the archive from SourceForge.net ⮷ is downloaded and unzipped [2] . Then run the script mod-install-debian.sh in the terminal in the directory pacpl.6.xx / extra to install missing modules that were not installed by, and then compile [3] the program. Unfortunately, however, the compilation aborts with an error under 18.04 . The package from Ubuntu 19.04 in version 6.1.0 can also be installed under 18.04 without any problems (see pacpl .)sudo apt-get build-dep pacpl

A notice!

Third-party software can endanger the system.

service

The basic command for conversion in the terminal [4] is:

pacpl --to FORMAT [OPTIONEN] DATEIEN_ODER_ORDNER

Options

Perl Audio Converter Options (selection)
Option Langform function
-t FORMAT --to FORMAT Desired output format (at the same time new file extension)
-r --recursive recursive conversion of files also in subdirectories
-p --preserve existing folder structure is also used for folders to be created
-o FORMAT --only FORMAT only the specified format is converted
-k --keep Files already in the desired format are skipped or moved directly
--dryrun Test run without actual changes to the files, very useful, especially with potentially dangerous options!
--normalize Standard normalization of audio files
--nopts OPTION Normalization, specifying options
--outdir Output directory to which the files will be written / moved
--outfile NAME Issue name; The standard is to keep the old name with the new file extension

To the individual encoders can be (depending on the format) special settings for quality, bit rate, number of channels, frequency, etc. The default settings, which uses pacpl without further information find in the file /etc/pacpl/pacpl.conf , There the information can also be changed as required (root rights [5] required!).

The manual page of the program provides a complete overview of the very extensive options .

example

The call

pacpl --to ogg -r -p /home/Musik --outdir /home/oggs

leads to the fact that the existing audio files (all supported formats are converted!) in the directory / home / music are written as .ogg files in the directory / home / oggs . For example, the data from / home / music / Rock / Krautrock would be stored as .ogg files in / home / oggs / Rock / Krautrock , since the conversion is recursive and the original directory structure is retained .

Kommadozeilen-CD-Ripper

CDs can be ripped according to the following pattern:

pacpl --rip all --to flac
pacpl --rip 1,3,9,15 --to flac

The first command rips all tracks on the inserted CD into .flac format, the second rips tracks 1, 3, 9 and 15.

CD ripper options
Option function
--rip OPTION Rip all ( all) or the tracks on the CD separated by commas (e.g. " 1,3,9,15")
--nocddb no CDDB query to create the tags (query is standard)
--noinput suppress interactive dialog for CDDB query (dialog is standard)
--device NAME Specify a device other than / dev / cdrom for the CD drive
--cdinfo Show CDDB information about the currently inserted CD
--nscheme STRING Specify information about the title issue. The following parameters are possible, which can be used instead of the standard "%ar-%ti"( artist - title. <Ending> ):
%ar Artist
%ti Songtitel
%tr Track number
%yr Publishing year
%ab Albenname

Example:

pacpl --rip all --to ogg --nscheme="%ar-%ab-(%tr)-%ti"

leads to the output of all CD titles according to the pattern artist-album- (track number) -title.ogg . Direct filing in the form of ARTIST / ALBUM / NR - TITEL is not possible, and the --outdiroption cannot be used for the ripper.

Use out of Amarok, Konquerer and Dolphin

Among the programs mentioned, the Perl Audio Converter can be used from the context menu , similar to audio convert for Nautilus . A in the desired file opens the context menu (several files can also be selected), in which the entry for the program can then be selected. The required information is then given there and the conversion is carried out. The version 4.0.5 available in the official Ubuntu sources is optimized for KDE4 (however, the implementation for Amarok 2 will not take place until one of the next releases).

Troubleshooting

Problems can arise when special characters appear in the file names. Commas in particular may cause the conversion to result in an error message such as

"Encoding has failed with the status: 256"

breaks off. In such a case, it may only help to remove the corresponding character from the file name.

However, you get the same error message if you have not installed the corresponding encoder. So: if you want to convert WAV to MP3 , for example , you need the LAME encoder or the lame package . The list of formats helps to find out which packages have to be installed . An encoder / decoder is listed there for the corresponding format. Now you just have to find the right package that contains this encoder / decoder and then install it.

However, this message can also indicate a defective output file. So if there are no special characters and all required codecs are installed, this could be the reason for the error message.

--Normalize option fails

Similar to K3b (section “normalize-audio-is-not-found”) there are problems with the function --normalize. The expected program is called under Ubuntu normalize-audiofrom the package of the same name. The K3b solution also works here; alternatively, a symbolic link with the command

sudo ln -s `which normalize-audio` /usr/local/bin/normalize

to be created.
