MaxKellermann commented on Oct 28, 2019

mpd doesn't fail. Your Linux kernel is configured in a way that binding to [::]:6600 (IPv6 wildcard) also binds to the IPv4 wildcard (via /proc/sys/net/ipv6/bindv6only). What you see is this log message:

exception: bind to '0.0.0.0:6600' failed (continuing anyway, because binding to '[::]:6600' succeeded): Failed to bind socket: Address already in use

That's a failure, but not a fatal failure, as the message states. MPD is operational.

Your mpd.socket failure is due to some other process already bound on port 6600. At this point, you need to investigate which process that is, but it is certainly not a MPD bug.
