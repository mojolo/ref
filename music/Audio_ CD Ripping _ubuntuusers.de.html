This overview describes the programs with which you can rip audio CDs under Linux and reduce the size of the files. The focus is on the popular formats MP3 , Vorbis (file extension .ogg ), Opus and FLAC .

FLAC plays a special role as it is a lossless format that is particularly suitable for archiving. Other formats can be generated from it later at any time, if required, without deteriorating the quality through generation loss.

In terms of lossy reduction, Opus and its predecessor Vorbis are superior to the popular MP3 in almost all respects. Both Opus and Vorbis compress more efficiently and are most importantly free formats . In contrast to MP3, this means that they can be used anywhere without licensing difficulties. Older hardware players ("MP3 players") rarely support Opus or Vorbis; some models can be remedied by installing the free Rockbox firmware . Newer players and smartphones usually support Opus and Vorbis as standard. A good overview of the advantages and disadvantages of the various audio formats can be found in the AudioHQ forum 🇩🇪.

If you want to copy CDs as true to the original as possible (e.g. for archiving), you will find more information in the Ripping Exact CD Image section .

Programme

Below is an alphabetical list of the programs available, sorted by desktop environment.

GNOME

Asunder

Asunder is a program based on GTK + for reading and converting audio CDs into another format. The program supports MP3 , Ogg-Vorbis , FLAC , WAV , Opus and Wavpack formats .

fre:ac

fre: ac (former name BonkEnc) is a free audio converter and audio CD ripper with support for various popular formats and an encoder optimized for multicore CPUs, with which audio CDs can also be read.

gnormalize

gnormalize is a kind of "Swiss Army Knife " for audio files. The program extracts audio CDs, adjusts the volume (normalization) and can convert between the formats MP3, MP4, MPC, Ogg-Vorbis, APE, FLAC and WAV . Anyone who has previously worked with grip will find an interesting alternative here.

Goobox

Goobox 🇬🇧 is a simple CD player and ripper for GNOME 3. To use Goobox, the following package must be installed [1] :

  • goobox (universe)

  • cdrdao (universe)

Package list to copy: 

sudo apt-get install goobox cdrdao

Or install with apturl , Link: apt: // goobox, cdrdao

After the CD has been read in, select "CD → Extract Tracks" from the menu . The desired format and the target directory can now be specified here. After clicking on "Extract" , the desired quality can be set with a slider.

Grip

Grip is a CD player and CD ripper for the GNOME desktop with very powerful functions and many detailed setting options. The further development was stopped in the meantime and the program was removed from the official package sources; it was resumed in 2016 (current version 3.4.0 from April 17, 2017). If you don't want to switch to an alternative program at any price, you can create the program yourself from the source code.

Rhythmbox

Rhythmbox (section “Audio CD”) is the standard music manager of the GNOME desktop environment and can also rip CDs. In addition, an album cover is downloaded and stored in the folder ~ / .cache / rhythmbox / covers .

ripperX

ripperX is particularly slim and thus represents an interesting alternative to Grip or Sound Juicer. It uses cdparanoiaand converts audio CDs into MP3 , Ogg Vorbis or FLAC . If available, a title list is also called up via an Internet database and the corresponding ID3v2 tags are automatically written.

Rubyripper

Rubyripper places particular emphasis on accurate copying. The developers oriented themselves to Exact Audio Copy (EAC), which is considered a reference among CD rippers in the Windows world.

Sound Juicer

Sound Juicer used to be an integral part of the GNOME desktop environment as a standard ripper . The program can be installed from the package sources. If you want to increase the readout speed, follow the tip mentioned below to deactivate the paranoia mode.

Xcfa

X Convert File Audio (Xcfa) is an application that offers a variety of functions under one hood. In addition to reading out the soundtrack from audio DVDs, ripping audio CDs and converting audio files, CD covers can also be created.

WHERE

Amarok

Although actually an audio player and contained in Kubuntu as a standard player, Amarok can also rip CDs. If a CD has been inserted, it will appear in the left program area under "Local Collection" . By clicking on the CD and then on the menu item "Copy to collection → Local collection" , a selection appears showing the format in which the pieces of music should be copied to the computer. In addition to being able to copy the tracks to the hard drive without converting them, the tracks can be automatically converted to Ogg-Vorbis , MP3 , WMA , FLAC or the "Apple LossLess" format. See Amarok .

Audex

Audex is an easy-to-use CD ripping program for KDE4. The output format is WAV , FLAC , Ogg-Vorbis , MP3 or MP4 / AAC, but any encoder that can be operated via the command line can be integrated.

K3b

K3b is primarily known as a burning program for KDE. However, it is not only possible to burn CDs , but also to read out CDs via the menu item "Extras → Read audio CD" .

Konqueror/Dolphin

In Dolphin and Konqueror it is possible to display the contents of audio CDs as virtual folders. To do this, enter the following in the address line:

audiocd:/

Such folders can simply be copied like normal folders, with the ripping and encoding process taking place automatically in the background. Cdparanoia is used for ripping and for coding lame(MP3) or oggenc(Ogg-Vorbis). The desired quality level can be adjusted under "System settings → Multimedia → Audio CDs" .

sound converter

soundKonverter is a program for converting audio files for the KDE desktop. CDs can also be ripped, and for certain formats there is also the option of calculating replay gain for the files.

Xfce, LXDE

Since Xfce and LXDE, just like GNOME, use the GTK + library for the graphical user interface, the following programs, already mentioned above, are suitable for these desktop environments:

  • Asunder

  • gnormalize

  • ripperX

  • Rubyripper

  • Xcfa

Terminal

abcde

abcde ("A Better CD Encoder") is a CD ripper for the command line with many setting options in the associated configuration file.

GStreamer

GStreamer is actually a complete multimedia framework that makes numerous basic routines available to other programs. However, it can also be used via the command line to read CDs. The following example uses Gstreamer-1.0 to create a file named tracktitel.ogg from the first track on the CD:

gst-launch-1.0 cdparanoiasrc device=/dev/cdrom track=1 ! decodebin ! vorbisenc ! filesink location=/PFAD/ZUM/ZIELVERZEICHNIS/tracktitel.ogg

The drive /dev/cdromspecification may have to be adjusted.

Perl Audio Converter

The extensive conversion tool Perl Audio Converter (section “Command Line CD Ripper”) also offers the option of ripping CDs into all supported formats.

whipper

whipper is a fork of the now discontinued program Archiv / morituri with almost identical operation, which is currently being further developed and maintained.

Others

jRipper

In jRipper 🇬🇧 is a program for reading CDs and converting audio files into different formats. The program is written in Java and can therefore be used under different operating systems.

Increase rip speed

All programs mentioned use the command line program cdparanoia . This runs error detection (called paranoia mode ) by default when reading out , which slows down the ripping - especially if there were problems. Since errors occur quite often when reading audio CDs, this procedure is justified, but if you still want / need to put speed over thoroughness, you can turn off paranoia mode in most programs.

To deactivate the paranoia mode for Sound Juicer, for example , an entry in the GNOME configuration editor must be changed. The entry "/ apps / sound-juicer / paranoia" must be opened there, where the entry "paranoia" is located. This must be changed from 8 to 0.

Paranoia mode can be deactivated globally under KDE . To do this, remove the check mark under "System settings → Advanced tab → Audio CDs → Use error correction when reading out the CD" .

Rip exact CD image

It is common practice to rip each track on an audio CD into a separate file. This is a disadvantage when songs merge, as is usually the case with live recordings, for example. Depending on the format and player, there may be disruptive interruptions that shouldn't actually exist. This problem is solved by ripping the CD as a whole image rather than track-by-track. In order to still be able to jump to the individual titles as usual while playing, a cuesheet (a map) must also be created.

This method can also be of general interest if you value a listening experience that is as true to the original as possible, because the audio player plays the file exactly as the CD player would have done with the disc. The image can later be broken down into the individual titles if necessary. Compatible burning programs can burn 1: 1 copies with the help of the cuesheet, in which the pauses between the songs also match those of the original disc.

The Windows program Exact Audio Copy (EAC) offers another mechanism: with the help of the MusicBrainz online database , the artist and title are clearly identified and then compared with a second online database called AccurateRip to produce an identical 1: 1 copy when Ensure ribs. If you are more interested in the topic, see Ripping Audio CDs for more information.

The following rippers can create an exact CD image:

  • abcde - Command line program

  • gnormalize

  • K3b - graphic ripper and burning program

  • whipper - Command line program

  • Rubyripper

  • sound converter

  • XCFA

In the same way, the following audio players can read out cuesheets:

  • Audacious

  • Banshee *

  • Clementine *

  • cmus (for the shell) *

  • DeaDBeeF (from external source)

Especially with players with a * database function , it makes no difference in terms of selection whether the songs are available individually or as a disc image with a cue sheet. For players without a database, the .cue file is usually opened and all the songs on the CD are displayed on it.

Cuesheet

A cue sheet contains the TOC ( T able O f C ontent = Table of Contents) of the copied CD. It was originally only developed for burning programs, but is increasingly being supported directly by audio players. The mandatory file with the extension .cue is usually located in the same folder as the associated audio file. Compatible players read the CUE file and then display the individual tracks as usual. Compatible burning programs structure the blank according to the cue sheet.

A CUE file contains plain text that can be displayed with any editor [3] and changed if necessary - however, the structure (with the preceding spaces) must be retained. A simple example of an audio CD that contains only two songs:

REM GENRE Gute-Laune-Musik

REM DISCID 12345
REM FREEDB_QUERY "gibts_gar_nicht"
PERFORMER "Linux Hausband"
TITLE "Drauf mit Linux!"
FILE "drauf_mit_linux.flac" WAVE
  TRACK 01 AUDIO
    TITLE "Endlich ein ordentliches System"
    PERFORMER "Linux Hausband"
    INDEX 01 00:00:00
  TRACK 02 AUDIO
    TITLE "Nie wieder bunte Fenster"
    PERFORMER "Linux Hausband"
    INDEX 01 04:40:03

  • Lines 1-5: Additional information about the audio CD, always recognizable from the above REM(technically unimportant).

  • Line 6 + 7: Information on the audio CD in the form of CD-Text , which is read out by compatible playback devices.

  • Line 8: The path to the ripped audio file, which of course has to be correct. The pending WAVEmust always be present, but has nothing to do with the audio format used.

  • From line 9: The individual tracks with track number ( TRACK), name ( TITLE), artist ( PERFORMER) and finally the start marker ( INDEX). TITLEand PERFORMERare burned again as CD-Text.

Rip image via script

The following packages are required [1] :

  • cdrdao (universe)

  • mkcue (universe)

  • vorbis-tools (universe)

Package list to copy: 

sudo apt-get install cdrdao mkcue vorbis-tools

Or install with apturl , Link: apt: // cdrdao, mkcue, vorbis-tools

procieedings

The following example script rips the CD as a whole and then converts the file to Vorbis:

enlarge
#!/bin/sh
cdrom=/dev/`readlink /dev/cdrom`

# if the line above doesnt work as expected, uncomment the
# following line and modify /dev/hdc to your needs:

#cdrom=/dev/hdc

cdrdao read-cd --with-cddb --device ATAPI:$cdrom --datafile cd.bin cd.toc
mkcue $cdrom > cd.cue
oggenc -o cdimage.ogg -r --raw-endianness 1 -q 6 cd.bin
rm cd.bin cd.toc

If the script does not automatically recognize the correct drive, the marked line must be commented out and adapted. Temporary files ( cd.bin and cd.toc ) are deleted again with the command in the last line. The content of the cue file should then be adjusted manually.

Break down the image into individual titles

It can happen that an audio player or a burning program does not support cuesheets. Depending on the audio format used, the image can be broken down as follows:

MP3 and Vorbis

The mp3splt program can read out cuesheets and cut the corresponding image directly. On top of that, the tags are even generated automatically. An example of an MP3 file [4] :

mp3splt -c album.cue album.mp3

Alternatively, this can also be done via a graphical user interface . First, select the audio file at the top, then the CUE file in the cddb & cue tab .

Other formats

Unfortunately, some effort has to be made here. The bchunk program is also able to cut images automatically. However, only WAVE is supported. The image must therefore be converted beforehand. For this you either use a specialized program or you use the decoder of the respective format. For example, for FLAC the command is:

flac -d album.flac

The path to the image ("FILE") must then be adapted in the CUE file by changing the file extension ( album.flac becomes album.wav ). The required program bchunk is installed from the package sources:

  • bchunk (universe)

Package list to copy: 

sudo apt-get install bchunk

Or install with apturl , Link: apt: // bchunk

The command to cut is:

bchunk -w album.wav album.cue NAME

At the end a name must be given, which is given to the individual titles and the associated track number. If you do not want to burn the files but convert them to a different format, they have to be tagged again afterwards , as the information from the cuesheet (artist, title name, etc.) cannot be adopted.

MusicBrainz database

Some CD rippers can access Internet databases and query information about a CD such as title, artist, year of publication etc. from there. The primary, freely usable database is MusicBrainz 🇬🇧. FreeDB used to be a free alternative, but this service was discontinued at the end of March 2020 🇩🇪.
