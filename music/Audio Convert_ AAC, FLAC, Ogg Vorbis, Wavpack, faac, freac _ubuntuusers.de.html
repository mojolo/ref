Converting audio files of a certain format to any other format is usually not a problem under Linux - it may only be difficult if the formats involved are exotic and / or formats (codecs) that are protected by license.

For the common formats such as Ogg / Vorbis , MP3 , FLAC and WAV there are numerous programs with a graphical user interface that make the conversion convenient. Programs with which CDs can be read (ripped) and saved in various formats are presented in the article Ripping CDs .

Caution!

Many audio formats work “lossy” and reduce the sound quality to a greater or lesser extent during conversion. If an audio file is converted from one lossy format to another (e.g. from MP3 to Vorbis ), the quality is always deteriorated due to generation loss . However, if the source format is lossless (e.g. FLAC ), there is no risk of this.

For the now outdated format Shorten ( .shn ), which is still used especially for bootlegs, instructions for installing the codec can be found under xcfa (section “External Encoder”) .

requirements¶

In order for the conversion to work without problems and to support as many file formats as possible, a few packages must first be installed [1] :

  • faac (multiverse)

  • faad (universe)

  • flac (universe)

  • lame (universe)

  • libmad0 (universe)

  • libmpcdec6 (universe)

  • mppenc (universe)

  • vorbis-tools (universe)

  • wavpack (universe)

Package list to copy: 

sudo apt-get install faac faad flac lame libmad0 libmpcdec6 mppenc vorbis-tools wavpack

Or install with apturl , Link: apt: // faac, faad, flac, lame, libmad0, libmpcdec6, mppenc, vorbis-tools, wavpack

Programs with a graphical user interface

Audacity

Audacity is a free, easy-to-use and very powerful audio editor and recorder for GNU / Linux, Mac OS X, Windows and other operating systems.

FF Multi Converter

FF Multi Converter is a clearly laid out graphical user interface which is suitable for converting audio, video, image and document files and supports most of the popular formats under one application. FFmpeg is used for conversion for audio / video files, unoconv for document files and ImageMagick for images. Written in Python and Qt , this application is subject to GPL3 and is therefore considered free software.

fre:ac

fre: ac (former name BonkEnc) is a free audio converter and audio CD ripper with support for various popular formats and an encoder optimized for multicore CPUs, with which audio CDs can also be read.

gnormalize

In addition to being an audio file conversion program, gnormalize is also a CD ripper and player. The program is also suitable for normalizing audio files.

mhWaveEdit

mhWaveEdit is a slim and easy-to-use audio editor that can convert audio files into all common formats.

MP3Gain

MP3Gain allows you to adjust the volume level of MP3 files without having to re-encode them. Since it is a pure command line program, several graphical interfaces have emerged over the years: MP3Gain-GUI .

mp3splt

With mp3splt you can cut files in the MP3 or Ogg / Vorbis format without having to re-encode them.

OggConvert

OggConvert is a program based on GStreamer-0.10 , which can be used up to and including Ubuntu 14.04 to convert various audio formats to Ogg / Vorbis.

Soundconverter

Soundconverter has a simple and self-explanatory interface. It supports the file types MP3, Ogg / Vorbis, FLAC, WMA and WAV. Various settings such as file format, quality (bit rate) of the target file, storage location, etc. can be selected via the partially German-language menu. Individual files or entire folders can then be selected for conversion using the corresponding buttons.

sound converter

soundKonverter (with K) is a program for converting audio files for the KDE desktop. You can also rip CDs. There is also the option of calculating replay gain for the files for certain formats .

traGtor

traGtor is a graphical user interface for converting various media formats using FFmpeg . traGtor has set itself the goal of providing a graphical user interface for the quick conversion of individual media files and, if possible, to provide the user with a large number of the most important functions and options without using the command line. Very instructive, as the options used are not hidden from the user - as is usual with other programs.

xcfa

xcfa (X Convert File Audio) converts among others WAV, FLAC, Monkey's Audio ( .ape ), WavPack, Ogg / Vorbis, M4a, AAC, Mpc and MP3. Very useful when converting audio files with outdated codecs.

File manager scripts

audio-convert

audio-convert 🇬🇧 is a script that is integrated into the Nautilus file manager and enables audio files to be easily converted using the context menu. The following packages must be installed:

  • nautilus-script-audio-convert (universe)

  • mplayer (universe)

  • nautilus-script-manager (universe )

Package list to copy: 

sudo apt-get install nautilus-script-audio-convert mplayer nautilus-script-manager

Or install with apturl , Link: apt: // nautilus-script-audio-convert, mplayer, nautilus-script-manager

The script must first be registered as a plugin for Nautilus. The following command in the terminal [3] is used for this :

nautilus-script-manager enable ConvertAudioFile

Finally, Nautilus must be restarted: 

nautilus -q

If you now select an audio file in Nautilus and click on it with the right mouse button, you can select the desired target format via "Scripts → ConvertAudioFile" . The formats MP3, Ogg / Vorbis, MPC, AAC and WAV are supported. Then you can specify whether you want to transfer the meta tags of the file to the new file. The quality can then be determined. The folder of the source file is always used as the target folder.

If nothing is visible in the context menu under "Scripts" after the installation , it could be because the script is in the wrong directory, as indicated in the article Nautilus / Scripts . Then it helps to copy the script from ~ / .gnome2 / nautilus-scripts / to ~ / .local / share / nautilus / scripts / .

audiokonverter

audio converter 🇬🇧 (note the k) is a script that can be used to convert audio files from the context menu of the KDE file manager Konqueror . The installation is very simple. The following two packages are installed by default under KDE 4; under other interfaces they have to be installed afterwards [1] :

  • konqueror (universe)

  • konsole (universe)

Package list to copy: 

sudo apt-get install konqueror konsole

Or install with apturl , Link: apt: // konqueror, konsole

Additional packages may be required during the installation under a different desktop environment than KDE.

Now download audiokonverter from its homepage 🇬🇧 ⮷ and unpack [4] the file in your own home directory . In a terminal [3] you switch to the folder you have just extracted and execute the following command for the installation:

sudo ./install.sh

The installation should be self-explanatory. If you click on a (supported) audio file in Konqueror with the right mouse button, you can select the target type of the file under "Actions → convert to" . The formats MP3, Ogg / Vorbis, M4a, FLAC and WAV are supported. During the conversion you will be asked for the desired target directory.

Command line programs

The following programs are executed in the terminal [3] .

swftools

The swfextract contained in swftools can be used to extract the audio stream from swf files (flash).

SoX

SoX is a very powerful audio converter and effects filter.

FLAC

FLAC is an audio codec with lossless compression ("Lossless Codec"). In contrast to lossy methods ("Lossy Codecs") - such as Vorbis or the widespread MP3 - when using FLAC it is possible to completely restore the audio material even after decoding without any loss of quality.

LAME

LAME is a cross-platform open source project for the generation of MP3 audio files, e.g. from (uncompressed) WAV files.

faac

faac is a codec that converts lossy to the AAC format. AAC is part of the MPEG-2 and MPEG-4 specification and is used on Apple devices, among others. Compared to MP3, AAC can generally achieve better quality compressed audio material with the same data rate.

The faac command of the same name expects the input file in .wav format by default . The command call for the encoding is then in the simplest case:

faac -o ausgabe.aac eingabe.wav

faac knows a number of options that can be looked up in the man page .

For example, to convert an existing MP3 file into AAC format on the command line, the MP3 file can be played using mpg123 and the data stream can be encoded to AAC using faac:

mpg123 -s input.mp3 | faac -P -X -w -o output.m4b -

vorbis-tools

The vorbis tools contain the Ogg / Vorbis encoder oggenc , the decoder oggdec , the Ogg / Vorbis command line player ogg123 , the ogginfo tool for displaying Ogg / Vorbis metadata, the Ogg / Vorbis file splitter vcut and the Metadata editor vorbiscomment .

To convert files from Wave format to Ogg / Vorbis format or vice versa, oggenc offers the following command options:

WAV to Ogg / Vorbis

oggenc -b 256 *.wav        #alle Wave-Dateien werden mit der Bitrate 256 kbps umgewandelt

Alternatively, the bit rate can be prior to the execution of the instruction specify the desired quality ( 0-9; 6is already very good):

oggenc stueck1.wav -q 6 -o track_001.ogg

Ogg / Vorbis to WAV

oggdec *.ogg               #alle Ogg-Dateien in WAV umwandeln

Further information can be found in the Vorbis article .

Perl Audio Converter

The Perl Audio Converter converts a variety of formats (it is also possible to extract the sound from video formats), optionally comes with a graphical user interface and can be integrated into various file managers.

Batch processing

Batch processing or batch operation is the name given to the conversion of several files in one go. This can be particularly useful on the command line.

Multiple files

If several files are to be converted at once, this can be done very easily with a single-line shell script. In the following general example, it is assumed that the foobar program will convert all files in the current directory with the extension .ALT to files with the extension .NEU in the appropriate format:

for i in *.ALT; do foobar "$i" -o "`basename "$i" .ALT` .NEU"; done;

If the respective program requires options, these must be added after the program name.

Examples:

  • Convert all WAV files to the Ogg / Vorbis format with quality level 6: 

    for i in *.wav; do oggenc "$i" -q 6 -o "`basename "$i" .wav` .ogg"; done;

  • mediumRecode MP3 files with the quality level. This can be useful, for example, if you want to make MP3 files with a high bit rate "smaller":

    for i in *.mp3; do lame --preset medium "$i" -o "`basename "$i" .mp3`_medium.mp3"; done;

    Since neither the file extension nor the file name change here, but you don't want to overwrite the original files, the trick is to remove the file extension ".mp3" and then not only add a new extension .mp3 , but also the _Medium is added to the file name . This does not change the syntax of the command.

  • Convert M4a encoded files to MP3 format with GStreamer :

    for i in *.m4a; do gst-launch-0.10 filesrc location="$i" ! faad ! audioconvert ! audio/x-raw-int,rate=44100,channels=2 ! lame name=enc mode=1 quality=2 vbr=0 bitrate=128 ! xingmux ! id3v2mux ! filesink location="`basename "$i" .m4a`.mp3"; done

    All source files with the extension .m4a are gst-launchexecuted here with the source defined as input. Then the file is decoded and converted with the codecs in joint stereo, standard quality and constant bit rate of 128 kBit / s and the file name is changed. The original files remain.

  • Convert OGG files to MP3: 

    for i in *.ogg; do oggdec -o - "$i" | lame -v - "`basename "$i" .ogg`".mp3; done

Multiple files with subdirectories

With the following line you can also easily go through whole directory trees, for example to convert all WAV files into the Ogg / Vorbis format:

find ./ -iname '*.wav' -print0 | xargs -0 -n1 -P4 oggenc -q6

The command must be executed in the folder that contains the other subfolders. The Ogg / Vorbis encoder creates new files with the same file name but new extension .ogg in the corresponding directories . With the option -Pnfor xargs ( n=1,2,3,...), noggenc processes can be started in parallel.

It works in a similar way for MP3 files with the lame encoder :

find ./ -iname '*.wav' -print0 | xargs -0 -n1 -P4 lame --preset standard

Here, too, -P4four processes are started with the option .

dir2ogg

Another very convenient option is offered by the dir2ogg program with the parameters -d(only the transferred directory) and -r(also all subdirectories). For example, if you want to create an Ogg / Vorbis counterpart in the ~ / Music / classic directory and all sub-folders for all audio files in MP3, WAV, M4a and WMA formats, the following command can be used:

dir2ogg -r ~/Musik/klassik/

Convert real audio

The Real Audio format is mainly used for streaming on the Internet, for example by some online radio stations . These streams can be saved with suitable tools . Real audio files can be played on Linux by many players, provided the correct codec has been installed. In many cases, however, one would like to convert files in "Real Audio" ( .ra ) format into a common format such as MP3 or WAV, for example to burn them on CD or to use them on a mobile player.

The conversion is not a problem under Linux. The programs MPlayer and FFmpeg can be used for this. Both use the codec library of the following package for conversion to another format:

  • libavcodec-extra ( universe , contains additional codecs)

Package list to copy: 

sudo apt-get install libavcodec-extra

Or install with apturl , Link: apt: // libavcodec-extra

This must therefore be installed [1] alongside the respective program .

The conversion with the help of FFmpeg is done with the help of the following command in the terminal [3] :

ffmpeg -i Quelldatei.rm Zieldatei.wav

That -istands for input . For the target file, in addition to the desired name, you must of course also specify the appropriate ending for the desired target format.

If you use the MPlayer , the corresponding line is a little more complicated:

mplayer -ao pcm:file=Zieldatei.wav Quelldatei.rm

Here the source file and the target file are specified the other way round. -aomeans audio output .

WMA

The proprietary Windows Media Audio format ( .wma ) from Microsoft can be found in some online shops, as it is one of the audio formats that are DRM-capable .

To convert all WMA files in a folder into MP3 files, the following commands can be used from the source folder:

MPlayer

To use the MPlayer (the newly created files are similar in size to the original file), the following command must be used:

for i in *.wma ; do mplayer -vo null -vc dummy -af resample=44100 -ao pcm:waveheader "$i" && lame --preset standard audiodump.wav -o "`basename "$i" .wma`.mp3"; done; rm -f audiodump.wav

ffmpeg

To use FFmpeg (the newly created MP3 files only have approx. 33% of the original WMA file size), the appropriate codecs must be installed. Then use the following command in the current directory:

for i in *.wma; do ffmpeg -i "$i" "${i%.*}.mp3"; done
