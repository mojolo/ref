KVM VirSH Command

1 Administration of virtual machines

1.1 List Virtual Machines

  • List the machines that are in run or suspend mode:
    Copy 
    [root@erpos1m ~]# virsh list
 Id    Name                           State
----------------------------------------------------
 2     iwastudio1                     running
 5     erpwssoap1                     running
  • List all machines:
    Copy 
    [root@erpos1m ~]# virsh list --all
 Id    Name                           State
----------------------------------------------------
 2     iwastudio1                     running
 6     erpwssoap1                     running
 7     erpwsweb1                      running
 -     erpwsint1                      shut off

1.2 Suspend a virtual machine

Set the virtual machine to pause mode.

Copy 
[root@erpos1m KVM]# virsh suspend erpwssoap1
Domain erpwssoap1 suspended

1.3 Resume a virtual machine

To resume a suspended virtual machine.

Copy 
[root@erpos1m KVM]# virsh resume erpwssoap1
Domain erpwssoap1 resumed

1.4 Shutting down a virtual machine

  • Shut down the virtual machine in a controlled manner.
    Copy 
    [root@erpos1m ~]# virsh shutdown erpwssoap1
Domain erpwssoap1 is being shutdown
  • Shut down the virtual machine forcibly due to a failure in the operating system.
    Copy 
    [root@erpos1m ~]# virsh destroy erpwssoap1
Domain erpwssoap1 destroyed

1.5 Start a virtual machine

When the virtual machine is not indicated the automatic start, or it is turned off through the destroy command.

Copy 
[root@erpos1m ~]# virsh start erpwssoap1
Domain erpwssoap1 started

1.6 Register a new domain using the definition XML file

To register a new domain starting from the XML file, the definition must be made and then started.

Copy 
[root@erpos1m ~]# virsh define erpwssoap1.xml
    Domain erpwssoap1 has been defined

1.7 Restart a virtual machine

Copy 
[root@erpos1m ~]#  virsh reboot erpwsweb1
    Domain erpwsweb1 is being rebooted

1.8 Enable automatic startup of a virtual machine

Set the automatic start of the virtual machine when the host where it is hosted is restarted.

Copy 
[root@erpos1m ~]# virsh autostart erpwssoap1
    Domain erpwssoap1 marked as autostarted

1.9 Disable automatic startup of a virtual machine

Copy 
[root@erpos1m ~]# virsh autostart --disable erpwssoap1
    Domain erpwssoap1 unmarked as autostarted

1.14 Add a new disk to a domain

Create the raw disk image for example:

Copy 
qemu-img create -f raw -o size=4G /images/sles11sp1.raw

Dump the configuration for the guest. In this example the guest is called Guest1 and the file is saved in the users home directory.

Copy 
virsh dumpxml erpwssoap1 > /tmp/erpwssoap1_backup.xml

Edit the configuration file:

Copy 
virsh
    edit erpwssoap1

Find the <disk> elements, these elements describe storage devices. The following is an example disk element:

Copy 
<disk type='file' device='disk'>
    <driver name='tap' type='aio'/>
    <source file='/var/lib/libvirt/images/Guest1.img'/>
    <target dev='vda'/>
    <address type='pci' domain='0x0000' bus='0x00' slot='0x07' function='0x0'/>
</disk>

Add the additional storage by duplicating or writing a new <disk> element. Ensure you specify a device name for the virtual block device attributes. These attributes must be unique for each guest configuration file. The following example is a configuration file section which contains an additional file-based storage container named FileName.img.

Copy 
<disk type='file' device='disk'>
    <driver name='qemu' type='raw' cache='none'/>
    <source file='/var/lib/libvirt/images/Guest1.img'/>
    <target dev='vda' bus='virtio'/>
    <address type='pci' domain='0x0000' bus='0x00' slot='0x07' function='0x0'/>
</disk>
<disk type='file' device='disk'>
    <driver name='qemu' type='raw' cache='none'/>
    <source file='/images/sles11sp1.raw'/>
    <target dev='vdb' bus='virtio'/>
</disk>

For then new disk entry avoid to put <address> data and ensure that the <target name> is not in use.

Then start the machine and you can see the new partitions whith the fdisk utility.

1.15 Attaching a disk image to a guests CD-ROM drive with virsh

Attach a disk image to a guest's CD-ROM drive:

Copy 
virsh attach-disk <GuestName> sample.iso hdc --type cdrom --mode readonly

Disk attached successfully Create an XML file to update a specific device. To detach, remove a line for the source device:

Copy 
<disk type='block' device='cdrom'>
  <driver name='qemu' type='raw'/>
  <target dev='hdc' bus='ide'/>
  <readonly/>
  <alias name='ide0-1-0'/>
  <address type='drive' controller='0' bus='1' unit='0'/>
</disk>

Detach a disk image by updating the device:

Copy 
virsh update-device <GuestName> guest-device.xml

Device updated successfully.

2 Storage Pools

2.1 List storage pools

  • List only active volumes:
    Copy 
    [root@erpos1 ~]# virsh pool-list
Name                 State      Autostart
-----------------------------------------
default              active     yes
  • List all volumes:
    Copy 
    [root@erpos1 ~]# virsh pool-list --all
Name                 State      Autostart
-----------------------------------------
default              active     yes
kvmfiles             inactive   yes

2.2 List the volumes created in a storage pool

Copy 
[root@erpos1 ~]# virsh vol-list kvmfiles
Name                 Path
-----------------------------------------
erpbalint1_d1.raw    /data2/KVM/erpbalint1_d1.raw
erpwsbatch1_d1.raw   /data2/KVM/erpwsbatch1_d1.raw
erpwsint1_d1.raw     /data2/KVM/erpwsint1_d1.raw

2.3 Add a storage pool

2.3.1 Define the new pool

The virsh pool-define-as command defines the new pool. It requires two arguments:

  1. Name of pool.
  2. Path of the directory where the KVM files will be stored.
Copy 
[root@erpos1m data2]# virsh pool-define-as kvmfiles dir - - - - "/data2/KVM"
Pool kvmfiles defined

2.3.2 Check if the pool has been created

Copy 
[root@erpos1m data2]# virsh pool-list --all
Name                 State      Autostart
-----------------------------------------
default              active     yes
kvmfiles             inactive   no

2.3.3 Create local directory

Copy 
[root@erpos1m data2]# virsh pool-build kvmfiles
Pool kvmfiles built

2.3.4 Start the storage pool

Copy 
[root@erpos1m data2]# virsh pool-start kvmfiles
Pool kvmfiles started

2.3.5 Enable automatic startup

Copy 
[root@erpos1m data2]# virsh pool-autostart kvmfiles
Pool kvmfiles marked as autostarted

2.3.6 Show Pool Information

Copy 
[root@erpos1m data2]# virsh pool-info kvmfiles
Name:           kvmfiles
UUID:           9a814360-fff6-d252-d31a-39dbc79db24c
State:          running
Persistent:     yes
Autostart:      yes
Capacity:       1.97 TiB
Allocation:     446.67 GiB
Available:      1.53 TiB

2.4 Delete a storage pool

2.4.1 Stopping the storage pool

Copy 
[root@erpos1m ~]# virsh pool-autostart kvmfiles2
Pool kvmfiles2 marked as autostarted

2.4.2 Delete the directory associated with the pool

You can optionally physically delete the directory assigned to the storage pool:

Copy 
[root@erpos1m ~]# virsh pool-delete kvmfiles2
Pool kvmfiles2 deleted

2.4.3 Disassociate the storage pool

Copy 
[root@erpos1m ~]# virsh pool-undefine kvmfiles2
Pool kvmfiles2 has been undefined
