katylava commented Jul 21, 2017

I ran into this as well after I installed https://github.com/zsh-users/zsh-history-substring-search. The problem was the order in which I was defining key bindings and sourcing the two plugins.

1. bind keys first, recommended order of sources

  • history search works properly and has syntax highlighting
  • syntax highlighting displays errors
bindkey '\e[A' history-substring-search-up
bindkey '\e[B' history-substring-search-down

source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh

2. bind keys first, switch order of sources

  • history search works, but doesn't have syntax highlighting
  • syntax highlighting doesn't display any errors
bindkey '\e[A' history-substring-search-up
bindkey '\e[B' history-substring-search-down

source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh
source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh

3. bind keys last, recommended order of sources

  • history search works and has syntax highlighting
  • syntax highlighting doesn't report any errors
source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh

bindkey '\e[A' history-substring-search-up
bindkey '\e[B' history-substring-search-down

So, 3 works, though I'd prefer to have my key bindings first, but it's not that big of a deal.

I ran into this as well after I installed https://github.com/zsh-users/zsh-history-substring-search. The problem was the order in which I was defining key bindings and sourcing the two plugins.
Thanks for the details @katylava!
**1. bind keys first, recommended order of sources** - history search works properly and has syntax highlighting - syntax highlighting displays errors ``` bindkey '\e[A' history-substring-search-up bindkey '\e[B' history-substring-search-down source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
At this point, history-substring-search-* are partially initialized, they have been referenced by bindkey but not yet created with `zle -N`. I remember fixing this, are you using latest master of z-sy-h?
source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh ``` **2. bind keys first, switch order of sources** - history search works, but doesn't have syntax highlighting - syntax highlighting doesn't display any errors ``` bindkey '\e[A' history-substring-search-up bindkey '\e[B' history-substring-search-down source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh ```
Here, by the time z-sy-h runs, all widgets are properly defined. What version of zsh are you using? z-sy-h works better with more recent zsh's.
**3. bind keys last, recommended order of sources** - history search works and has syntax highlighting - syntax highlighting doesn't report any errors ``` source /usr/local/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh source /usr/local/share/zsh-history-substring-search/zsh-history-substring-search.zsh bindkey '\e[A' history-substring-search-up bindkey '\e[B' history-substring-search-down ``` So, 3 works, though I'd prefer to have my key bindings first, but it's not that big of a deal.
Try your option (1) but also add `zle -N history-substring-search-up; zle -N history-substring-search-down` before either `source` command?
